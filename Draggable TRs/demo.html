Draggable <tr> elements

A browser limitation prevents Scriptaculous draggables from being applied to table rows. This can be a problem when tables are being used in a semantically-correct manner, yet rows need to be dragged. The following patch to Scriptaculous attempts to circumvent this limitation by slyly cloning the containing table and creating a twin sibling of it, containing the row to be dragged.

Right now, it only works with ghosting turned on.
